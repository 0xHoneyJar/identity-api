```bash
# Unconditional sections (C + E):

# Full suite (Sections A/B/D/F require real PG):
```

(scratch-DB safety gate). Without `TEST_DATABASE_URL`, Sections A, B, D, F

### G-2.1 — 5-path convergence (single user_id from wallet/discord/(world,nym))

**Tested by:**

| File | Line | Test name |
|------|------|-----------|
| `src/api/__tests__/phase1-goal-validation.test.ts` | 512 | `G-2 unified walk: 1 human / 2 wallets / 2 nyms / 1 discord → ALL 5 lookups return SAME user_id` |

**Walk-through of the load-bearing test:**

1. Generate viem keypair A → wallet `W1`; run /v1/auth/challenge +
   /v1/auth/verify (SIWE) → identity-api mints user `U`, links `W1` primary.
2. Generate viem keypair B → wallet `W2`; spine.linkWallet(U, W2, primary=false).
3. spine.linkAccount(U, 'discord', `discord-test-user-7777`).
4. spine.claimNym(U, 'thj', 'honeykeeper').
5. spine.claimNym(U, 'mibera', 'fullshape').
6. The 5 LOAD-BEARING ASSERTIONS via SDK:
   - `client.resolve.byWallet(W1).user_id === U`
   - `client.resolve.byWallet(W2).user_id === U`
   - `client.resolve.byAccount('discord', 'discord-test-user-7777').user_id === U`
   - `client.resolve.byNym('thj', 'honeykeeper').user_id === U`
   - `client.resolve.byNym('mibera', 'fullshape').user_id === U`
7. Set-based convergence assertion: `new Set([5 user_ids]).size === 1`.
8. DB invariant: `SELECT COUNT(*) FROM users === 1`.
9. Composite `client.identity.get(U)` reflects the full graph: 2 wallets,
   1 linked_account, 2 world_identities, primary=W1.

**Tested by (three independent detectors):**

| `src/api/__tests__/phase1-goal-validation.test.ts` | 260 | `e2e: POST /v1/auth/verify with scheme='dynamic_user_id' → 400 (Zod enum rejection)` | Runtime defense — Zod boundary |
| `src/api/__tests__/phase1-goal-validation.test.ts` | 287 | `source-tree assertion: zero live-path files import @dynamic-labs/* (direct file scan)` | Independent grep — defense-in-depth |
| `scripts/__tests__/check-dynamic-quarantine.test.ts` | (entire file) | Original quarantine script tests (current-tree + synthetic-violation) | Original gate authoring (T1.7) |
| `src/api/__tests__/auth-bridge-quarantine.test.ts` | (entire file) | Bridge registry: dynamic has usableInLivePath=false | Structural — registry contract |

**Empirical quarantine output (live tree, 2026-05-24):**

```
$ bash scripts/check-dynamic-quarantine.sh
OK: zero @dynamic-labs/* live-path imports detected.
$ echo $?
0
```

**Three layers of defense:**

1. **Zod schema (compile + runtime):** `VerifyReqSchema.scheme` is
   `z.enum(["siwe", "eip191"])` — `dynamic_user_id` is statically
   inadmissible. Test at line 260 proves the runtime rejection.
2. **Bridge registry flag (runtime):** even if a future PR added
   `dynamic_user_id` to the Zod enum, the route handler at
   `src/api/routes/auth.ts:300` checks `bridge.usableInLivePath` and
   rejects. Test at `auth-bridge-quarantine.test.ts:44` asserts the
   dynamic bridge's flag is `false`.
3. **Source-grep gate (CI):** `scripts/check-dynamic-quarantine.sh`
   walks every live-path TS file and rejects any `import @dynamic-labs/*`
   shape. Run inline as test at line 241; PLUS an independent grep
   at line 287 as defense-in-depth.
